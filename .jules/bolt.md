## 2024-05-16 - Fixed N+1 query in Note Search
**Learning:** Room DAOs can easily handle batch fetching using `IN (:ids)` lists, which drastically reduces database roundtrips. Inside `searchNotes` in `NoteRepository.kt`, mapping over each result and querying the database line-by-line led to massive N+1 slowdowns as the search hit list grew.
**Action:** Replaced O(N) database queries with O(1) batch queries using `getByIds` and mapped the results in memory using `associateBy`. Added an early return `if (recognitionHits.isEmpty()) return@map emptyList()` to avoid SQLite `IN ()` syntax errors. Next time, always check loops in repository layers for N+1 query patterns.
## 2026-03-23 - Replaced hypot with custom math
**Learning:** Math.hypot enforces Double allocation internally and checking overflow/underflow, which brings a high overhead in tight hot loops inside UI 120Hz/60Hz touch event handling (such as pan gestures in Android). When checking velocity threshold or distances inside Canvas bounding bounds, squaring operations and computing Euclidean squared limits performs significantly better. Also, sequence operations combined with `.flatten()` avoid creating intermediate collections compared to summing and `.distinctBy`.
**Action:** Instead of `hypot(dx, dy)`, manual threshold comparisons were migrated to `dx*dx + dy*dy > threshold*threshold`, and manual distance equations used `sqrt(dx*dx + dy*dy)` inside `InkCanvasTransformTouch.kt`.
