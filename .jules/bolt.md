## 2024-05-16 - Fixed N+1 query in Note Search
**Learning:** Room DAOs can easily handle batch fetching using `IN (:ids)` lists, which drastically reduces database roundtrips. Inside `searchNotes` in `NoteRepository.kt`, mapping over each result and querying the database line-by-line led to massive N+1 slowdowns as the search hit list grew.
**Action:** Replaced O(N) database queries with O(1) batch queries using `getByIds` and mapped the results in memory using `associateBy`. Added an early return `if (recognitionHits.isEmpty()) return@map emptyList()` to avoid SQLite `IN ()` syntax errors. Next time, always check loops in repository layers for N+1 query patterns.## 2024-04-04 - Replaced hypot with manual sqrt in touch hot loops
**Learning:** Using `Math.hypot` or `kotlin.math.hypot` inside fast-firing touch or gesture loops carries unnecessary overhead from overflow/underflow checks and `Double` casting. Because standard screen sizes and touch velocities easily fit inside regular 32-bit `Float` limits without overflow, doing `sqrt(dx*dx + dy*dy)` on pure floats is significantly faster.
**Action:** Avoid `hypot()` in UI coordinate and touch velocity calculations. Always opt for `sqrt(dx*dx + dy*dy)` using `Float` directly when bounds are known to be safe.
