## 2024-05-16 - Fixed N+1 query in Note Search
**Learning:** Room DAOs can easily handle batch fetching using `IN (:ids)` lists, which drastically reduces database roundtrips. Inside `searchNotes` in `NoteRepository.kt`, mapping over each result and querying the database line-by-line led to massive N+1 slowdowns as the search hit list grew.
**Action:** Replaced O(N) database queries with O(1) batch queries using `getByIds` and mapped the results in memory using `associateBy`. Added an early return `if (recognitionHits.isEmpty()) return@map emptyList()` to avoid SQLite `IN ()` syntax errors. Next time, always check loops in repository layers for N+1 query patterns.

## 2024-05-24 - Replaced hypot() calls in Android touch and animation loops
**Learning:** Using `kotlin.math.hypot()` inside performance-critical hot loops (like touch event handlers `MotionEvent` and frame rendering loops like `withFrameNanos`) creates unnecessary overhead. `hypot()` computes `sqrt(x*x + y*y)` but carries built-in overflow/underflow checking. Also, on Float inputs, it forces `.toDouble()` conversions.
**Action:** Instead of `hypot`, manually calculate the square root using `sqrt((x*x + y*y).toDouble())`, or even better, eliminate the square root entirely by comparing squared magnitude against a squared threshold: `(x*x + y*y) > (threshold*threshold)`. Always look for math functions containing hidden overflow checks or extra casts in high-frequency loops.
