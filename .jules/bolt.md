## 2024-05-16 - Fixed N+1 query in Note Search
**Learning:** Room DAOs can easily handle batch fetching using `IN (:ids)` lists, which drastically reduces database roundtrips. Inside `searchNotes` in `NoteRepository.kt`, mapping over each result and querying the database line-by-line led to massive N+1 slowdowns as the search hit list grew.
**Action:** Replaced O(N) database queries with O(1) batch queries using `getByIds` and mapped the results in memory using `associateBy`. Added an early return `if (recognitionHits.isEmpty()) return@map emptyList()` to avoid SQLite `IN ()` syntax errors. Next time, always check loops in repository layers for N+1 query patterns.
## 2025-02-12 - Optimized hot loops touch math in InkCanvasTransformTouch and NoteEditorScreen
**Learning:** In performance-critical hot loops (e.g., canvas geometry calculations, erasing logic, touch gesture velocity tracking), using expensive math operations like `kotlin.math.hypot` can introduce unnecessary overhead from overflow/underflow checking, type conversions (Double), and scaling. Since UI coordinates and velocity values are well within standard `Float` or `Double` ranges, mathematical equivalents are perfectly safe.
**Action:** Replace `hypot` with simple square distance magnitude comparisons (e.g., `(vx * vx + vy * vy) > (threshold * threshold)`) or manual direct standard `sqrt` calculations (e.g., `sqrt(dx*dx + dy*dy)`) to eliminate function overhead and unnecessary casting in touch interaction loops.
