## 2024-05-16 - Fixed N+1 query in Note Search
**Learning:** Room DAOs can easily handle batch fetching using `IN (:ids)` lists, which drastically reduces database roundtrips. Inside `searchNotes` in `NoteRepository.kt`, mapping over each result and querying the database line-by-line led to massive N+1 slowdowns as the search hit list grew.
**Action:** Replaced O(N) database queries with O(1) batch queries using `getByIds` and mapped the results in memory using `associateBy`. Added an early return `if (recognitionHits.isEmpty()) return@map emptyList()` to avoid SQLite `IN ()` syntax errors. Next time, always check loops in repository layers for N+1 query patterns.
## 2025-04-03 - Avoid allocating intermediate collections and Path objects in Compose DrawScope
**Learning:** In Jetpack Compose, the `DrawScope` loop executes frequently (often 60+ times per second). Allocating new objects like `Path()` or using Kotlin collection functions that create intermediate collections (like `partition`) inside drawing or rendering blocks generates significant garbage collection pressure. This leads to frame drops and UI jank.
**Action:** When drawing paths in a loop, hoist the `Path` allocation outside the drawing loop (e.g. `val path = remember { Path() }`), reuse it inside using `.reset()`, and aggregate multiple sub-paths to draw them with a single `drawPath` call. Instead of creating intermediate lists with functions like `partition`, iterate multiple times using `forEach` with conditions.
