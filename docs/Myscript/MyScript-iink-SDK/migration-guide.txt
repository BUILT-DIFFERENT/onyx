Migration guide

This page lists the key points to pay attention to when migrating existing code from to the latest MyScript iink SDK APIs.
Migrating to iink SDK 4.️3 native APIs
GestureHandler has a new method onShapeOnHold

For WPF and UWP, if you have a GestureHandler, you must add an implementation for the onShapeOnHold callback.
Old .iink files (iink SDK 1.3 or earlier) are no longer compatible

MyScript iink SDK 4.3 cannot open a ContentPackage that was generated with iink SDK 1.3 or earlier. However, if a ContentPackage was created with iink SDK 1.3 (or earlier) and then opened and saved with a later version, iink SDK 4.3 will still be able to open it.
SK compile options have changed

The available SK options for the RecognitionAssetsBuilder compile method have been updated to be more generic: Math Disabled Subset and Math Enabled Subset have been replaced with Disabled Subset and Enabled Subset.
MyScriptPrediction library removal

The MyScriptPrediction library is no longer shipped since 4.0.0 because it is no longer used. However, as of 4.1.0, this library is no longer compatible with other libraries, so if is still in your environment, you may encounter errors when loading the iink libraries and your application may crash at runtime. To avoid problems when upgrading to 4.3, search for any file named *MyScriptPrediction.* in your application environment and delete it.
